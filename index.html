Getting started

Pick the route that matches your integration

The Formlabs APIs are REST-style HTTP APIs. Use the Local API when the workflow needs PreFormServer and local files; use the Web API when the workflow needs Dashboard-connected devices and account data.

Local

Automate job preparation on a workstation

  1. Download PreFormServer for macOS or Windows from the Formlabs API downloads page.
  2. Start it with an explicit port, commonly 44388.
  3. Wait for READY FOR INPUT before sending requests.
  4. Use absolute local file paths when loading or saving models and scenes.
PreFormServer.exe --port 44388

curl http://localhost:44388/
curl http://localhost:44388/scene/
Cloud

Monitor Dashboard-connected printers

  1. Create Formlabs Dashboard application credentials.
  2. Request an OAuth access token with client_credentials.
  3. Send the token as Authorization: Bearer ....
  4. Respect rate limits and refresh tokens before they expire.
curl -X POST https://api.formlabs.com/developer/v1/o/token/ \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" \
  -d "client_id=$CLIENT_ID" \
  -d "client_secret=$CLIENT_SECRET"

curl https://api.formlabs.com/developer/v1/printers/ \
  -H "Authorization: Bearer $ACCESS_TOKEN"

HTTP fundamentals

Use predictable requests and JSON responses

Both APIs expose resources over HTTP. The Local API defaults to http://localhost:44388; the Web API defaults to https://api.formlabs.com. Successful responses use conventional 2xx status codes, client errors use 4xx, and server errors use 5xx.

Local state

Scenes are cached while PreFormServer runs. Prefer explicit scene IDs over deprecated default-scene endpoints.

Async work

Long-running Local API operations can use ?async=true and then be polled through operations endpoints.

Web limits

The Web API documents per-IP and per-user limits. Handle 429 responses with the provided retry timing.

Python SDK

Install the SDK packages from the Formlabs repo

The SDK repository includes a minimal Local API helper, generated Local and Web clients, and runnable examples. The generated clients give typed request models; the minimal local package leaves request formatting in your code. 

git clone https://github.com/Formlabs/formlabs-api-python.git
cd formlabs-api-python

python3 -m venv .venv
source .venv/bin/activate
python3 -m ensurepip --default-pip

python3 -m pip install -e local-api/minimal-lib
python3 -m pip install -e local-api/lib
python3 -m pip install -e web-api/lib
import formlabs_local_api as formlabs

configuration = formlabs.Configuration(
    host="http://localhost:44388",
)

with formlabs.ApiClient(configuration) as api_client:
    api = formlabs.APIInfoApi(api_client)
    print(api.get_api_version())
